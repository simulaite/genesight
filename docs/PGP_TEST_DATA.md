# PGP Test Data

## What is the Personal Genome Project (PGP)?

The [Harvard Personal Genome Project](https://pgp.med.harvard.edu/) (PGP) is a long-term
research study at Harvard Medical School that invites volunteers to publicly share their
genomic, health, and trait data. Participants explicitly consent to making their data fully
public under the **CC0 1.0 Universal (Public Domain Dedication)** license. This means the
data can be freely used for any purpose, including software testing, without restriction.

**Source:** <https://my.pgp-hms.org/public_genetic_data>

## Why We Use PGP Data

GeneSight uses synthetic test fixtures for unit tests (in `tests/fixtures/`). However,
synthetic data cannot fully exercise real-world parsing edge cases such as:

- Realistic variant distributions across chromosomes
- Actual no-call (`--`) patterns from genotyping arrays
- Internal 23andMe probe IDs (`i`-prefixed identifiers)
- Realistic heterozygosity rates and allele distributions
- Mitochondrial and Y-chromosome haploid calls
- The difference between 23andMe v4 and v5 chip layouts

PGP data gives us **real, representative genotyping data** for integration testing without
any ethical or legal concerns, since participants have explicitly consented to full public
release under CC0.

### Key Differences Between v4 and v5

| Property | v4 (huAE4518) | v5 (hu09B28E, hu50801B) |
|----------|---------------|------------------------|
| Total variants | ~602K | ~631K-644K |
| Internal IDs (`i*`) | ~48K (8.0%) | ~17K (2.7%) |
| Heterozygosity rate | ~28.6% | ~16.0-16.5% |

The v5 chip has more total variants but fewer internal (non-rsID) probes. The apparent
heterozygosity difference is likely due to the biological sex of the participants (huAE4518
is female, so the X chromosome is diploid and contributes heterozygous calls, while the
v5 participants appear to be male with haploid X/Y).

## File Format

All files use the standard 23andMe raw data export format:

```
# This data file generated by 23andMe at: <timestamp>
# ...header comments...
# rsid	chromosome	position	genotype
rs548049170	1	69869	TT
rs9283150	1	565508	AA
i701234	MT	5678	G
```

- **Delimiter:** Tab-separated
- **Columns:** rsid, chromosome, position, genotype
- **Genotypes:** Two characters for diploid (e.g., `AA`, `AG`), one character for haploid
  (MT, male X/Y), or `--` for no-call
- **IDs:** Either `rs`-prefixed (dbSNP rsIDs) or `i`-prefixed (23andMe internal probe IDs)
- **Reference build:** GRCh37 / hg19
- **Comment lines:** Start with `#`

## How to Download

The files are hosted on PGP's Arvados storage. To download them:

```bash
# Create the directory
mkdir -p tests/fixtures/pgp

# hu09B28E (23andMe v5, 5.5 MB compressed)
curl -L -o /tmp/pgp_hu09B28E.zip \
  "https://46de496072e6785dbe2aed5aa92fa119-101.collections.ac2it.arvadosapi.com/_/genome_Joshua_Yoakem_v5_Full_20250127054538.zip"
unzip -p /tmp/pgp_hu09B28E.zip > tests/fixtures/pgp/hu09B28E_23andme_v5.txt

# huAE4518 (23andMe v4, 5.0 MB compressed)
curl -L -o /tmp/pgp_huAE4518.zip \
  "https://5048e1a37aefb72d900d62d091fab991-103.collections.ac2it.arvadosapi.com/_/genome_Marika_Forsythe_v4_Full_20240826181111.zip"
unzip -p /tmp/pgp_huAE4518.zip > tests/fixtures/pgp/huAE4518_23andme_v4.txt

# hu50801B (23andMe v5, 5.6 MB compressed)
curl -L -o /tmp/pgp_hu50801B.zip \
  "https://my.pgp-hms.org/user_file/download/4162"
unzip -p /tmp/pgp_hu50801B.zip > tests/fixtures/pgp/hu50801B_23andme_v5.txt

# Clean up
rm -f /tmp/pgp_hu*.zip
```

## How to Use for Testing

### Integration tests

These files can be used to validate that the parser handles real-world data correctly:

```rust
#[test]
#[ignore] // Requires PGP data download
fn test_parse_pgp_23andme_v5() {
    let data = std::fs::read_to_string("tests/fixtures/pgp/hu09B28E_23andme_v5.txt")
        .expect("PGP data not found — run download script first");
    let variants = genesight_core::parser::parse_23andme(&data).unwrap();
    assert!(variants.len() > 600_000);
}
```

Mark integration tests with `#[ignore]` so they only run when explicitly requested:

```bash
# Run only PGP integration tests
cargo test -- --ignored

# Run all tests including PGP
cargo test -- --include-ignored
```

### CLI smoke tests

```bash
cargo run -p genesight-cli -- analyze tests/fixtures/pgp/hu09B28E_23andme_v5.txt
```

### Benchmarking

The PGP files are useful for performance benchmarks since they represent real-world file
sizes (~16 MB, ~630K variants):

```rust
#[bench]
fn bench_parse_real_23andme(b: &mut Bencher) {
    let data = std::fs::read_to_string("tests/fixtures/pgp/hu09B28E_23andme_v5.txt")
        .unwrap();
    b.iter(|| genesight_core::parser::parse_23andme(&data));
}
```
